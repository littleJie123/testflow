# 需求
TestCase增加目录，目录下可以有TestCase，目录下可以有目录，也可以有其他TestCase。
服务端代码见src目录下directory和TestRunner。

## 目前实现
服务端代码已经写好，只需要增加客户端。

## 客户端涉及文件
client目录下的index.html 和 detail.html。

### index.html
1. 增加path变量，用于存储当前目录路径。默认为空字符串。  
2. 查询TestCase列表的时候，（访问listTestCase）需要传path参数。
3. 显示查询结果的时候，区分普通的TestCase和目录（type = 'directory'）。
4. TestCase的操作保持不变。目录只有一个“进入”操作，点击进入的时候，改变path参数，重新请求listTestCase,并重新渲染查询结果列表。在非根目录下，查询结果中默认有一个“上一级”目录，点击可以返回上一级目录。
